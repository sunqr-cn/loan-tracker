# 还款账户功能 - 产品设计

## 功能概述

新增"还款账户"功能，模拟银行卡管理还款资金：
- 账户余额管理（转入/转出）
- 交易流水记录
- 自动扣款（还款日自动扣除月供）

---

## 一、数据结构设计

### 1.1 新增数据字段

在现有 API 数据结构中新增 `repaymentAccount` 对象：

```typescript
interface RepaymentAccount {
  balance: number;              // 账户余额
  transactions: Transaction[];  // 交易流水
}

interface Transaction {
  id: string;                   // 交易ID（时间戳）
  date: string;                 // 交易日期 YYYY-MM-DD
  type: 'deposit' | 'withdraw' | 'repayment';  // 转入/转出/自动扣款
  amount: number;               // 金额（正数）
  balanceAfter: number;         // 交易后余额
  note?: string;                // 备注（如：第X期还款）
}
```

## 三、自动扣款逻辑

### 3.1 扣款时机

在 `reminder/recalc.js` 的自动任务中：
- 当还款日 <= 今天 时，标记该期为"已还"
- 如果账户余额 >= 月供金额，自动扣除并记录流水
- 如果余额不足，标记为"待充值"状态，不扣款

### 3.2 扣款逻辑伪代码

```javascript
// 在 recalc.js 中
function processAutoRepayment(data, schedule) {
  const account = data.repaymentAccount || { balance: 0, transactions: [] };
  const today = todayStr();
  
  for (const item of schedule) {
    // 找到今天到期的还款
    if (item.date === today && !item.paid) {
      if (account.balance >= item.monthlyPayment) {
        // 余额充足，自动扣款
        account.balance -= item.monthlyPayment;
        account.transactions.push({
          id: Date.now().toString(),
          date: today,
          type: 'repayment',
          amount: item.monthlyPayment,
          balanceAfter: account.balance,
          note: `第${item.period}期自动还款`
        });
        item.paid = true;
        console.log(`自动扣款成功：第${item.period}期 ¥${item.monthlyPayment}`);
      } else {
        // 余额不足
        console.log(`账户余额不足，需要 ¥${item.monthlyPayment}，当前 ¥${account.balance}`);
        // 不标记为已还，等待用户充值后手动处理
      }
    }
  }
  
  return account;
}
```

### 3.3 余额不足处理

- 在首页 Dashboard 显示提醒："还款账户余额不足，请充值"
- 在还款账户页面显示红色提示
- 可选：发送邮件提醒充值

## 七、边界情况处理

| 场景 | 处理方式 |
|------|----------|
| 首次使用，无账户数据 | 自动初始化空账户（余额0，空流水） |
| 转出金额 > 余额 | 提示"余额不足"，禁止操作 |
| 自动扣款时余额不足 | 不扣款，显示提醒，等待充值 |
| 利率变更后月供变化 | 按新月供扣款 |
| 提前还款后 | 不影响账户，用户自行管理 |
